│ 1. ANALYZER RÉALISTE (difficulty-v2.js)                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│ ✅ Algorithme professionnel basé sur 7 critères                            │
│    1️⃣ Modules PeopleNet impactés (0-2 pts)                                 │
│    2️⃣ Mots-clés de risque (0-2 pts)                                        │
│    3️⃣ Qualité du scénario (0-2 pts)                                        │
│    4️⃣ Longueur description (0-2 pts)                                       │
│    5️⃣ Nombre de commentaires (0-2 pts)                                     │
│    6️⃣ Ticket réouvert (0-2 pts)                                            │
│    7️⃣ Priorité Mantis (0-2 pts)                                            │
│                                                                             │
│ ✅ Score brut (0-14) → Score final (1-5)                                   │
│    • 0-2  → 1/5 (Faible)   🟢                                              │
│    • 3-4  → 2/5 (Faible)   🟢                                              │
│    • 5-6  → 3/5 (Moyenne)  🟠                                              │
│    • 7-8  → 4/5 (Élevée)   🔴                                              │
│    • 9+   → 5/5 (Élevée)   🔴                                              │
│                                                                             │
│ ✅ Modules PeopleNet détectés :                                            │
│    Paie, Absence, Planning, RH, Contrat, Règlements, Formation...          │
│                                                                             │
│ ✅ Mots-clés de risque :                                                   │
│    régression, incohérence, multi-client, erreur paie, corruption,         │
│    perte de données, blocage, DSN, URSSAF...                               │
│                                                                             │
│ ✅ Génération automatique :                                                │
│    - Résumé du ticket                                                      │
│    - Risques détectés                                                      │
│    - Recommandations d'action                                              │

│ 3. API REST (smartticket.routes.js)                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│ ✅ GET /smartticket/analyze?url=<ticket_url>                               │
│                                                                             │
│ Fonctionnalités :                                                          │
│ • Extraction automatique de l'ID depuis l'URL                              │
│ • Mapping ID → Ticket réaliste                                             │
│ • Création ticket générique si URL inconnue                                │
│ • Retour JSON complet avec critères détaillés                              │
│                                                                             │
│ Format de réponse :                                                        │
│ {                                                                           │
│   "ticketId": "111525",                                                     │

│     "riskWords": ["incohérence", "multi-client"],                           │
│     "missingScenario": false,                                               │
│     "descriptionSize": 1300,                                                │
│     "commentsCount": 4,                                                     │
│     "reopened": 0,                                                          │
│     "priority": "high"                                                      │
│   },                                                                        │
│   "modules": ["Absence", "Planning"],                                       │
