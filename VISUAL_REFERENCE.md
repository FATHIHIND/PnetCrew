## 🎯 États de la section SmartTicket

### État 1 : Masqué (par défaut)
- Section `#difficultySection` a `display: none`
- Pas d'appel API ou erreur réseau
- Pages chrome:// ou extension://

### État 2 : Visible avec données
- Section `#difficultySection` a `display: block`
- API a retourné `{ difficultyScore, modules, risks }`
- Badge coloré selon le score
- Barre remplie proportionnellement
- Listes peuplées

### État 3 : Visible avec valeurs par défaut
- API retourne score 2.0 par défaut
- Modules : ["Général"]
- Risques : ["Scénario manquant"]

## 🧪 Exemples de données

### Exemple 1 : Ticket simple
```json
{
  "difficultyScore": 1.5,
  "modules": ["Général"],
  "risks": ["Scénario manquant"]
}
```
**Résultat** : Badge VERT, 1 segment, peu de risques

### Exemple 2 : Ticket moyen
```json
{
  "difficultyScore": 3.0,
  "modules": ["Planning"],
  "risks": ["Scénario manquant", "Documentation incomplète"]
}
```
**Résultat** : Badge ORANGE, 3 segments

### Exemple 3 : Ticket critique
```json
{
  "difficultyScore": 4.5,
  "modules": ["Absence", "Planning", "Paie"],
  "risks": [
    "Cross-module",
    "Scénario manquant",
    "Risque d'intégrité des données",
    "Plusieurs clients affectés"
  ]
}
```
**Résultat** : Badge ROUGE, 5 segments, multiples risques

## 🎬 Animation et transitions

### Badge
```css
transition: all 0.3s;
```
Change de couleur fluidement lors du changement de score

### Segments
```css
transition: all 0.3s ease;
animation: pulse-segment 1.5s infinite;
```
Effet de pulsation sur les segments actifs

### Badges de modules
```css
transition: all 0.2s;
/* Au hover : */
transform: translateY(-2px);
box-shadow: 0 4px 8px rgba(59, 130, 246, 0.4);
```
Élévation au survol

## 📐 Architecture du code

```
extension/
  popup.html         ← Section #difficultySection ajoutée
  popup.css          ← Styles .difficulty-* ajoutés
  popup.js           ← Fonctions loadTicketDifficulty() ajoutées

server/
  smartticket/
    routes/
      smartticket.routes.js  ← GET /analyze?url=... ajouté
```

## ✅ Points de validation

1. **Badge change de couleur** selon le score (1-2 vert, 3 orange, 4-5 rouge)
2. **Barre se remplit** proportionnellement (score 4 → 4 segments actifs)
3. **Modules affichés** en badges bleus
4. **Risques listés** avec icônes ⚠️
5. **Section masquée** si pas de données ou erreur
6. **Compatible** avec les autres sections (résumé IA, checklist, etc.)
